Icons

Icons are tiny signage--simple, bold, and readable like symbols on old equipment. They should feel hand-drafted but precise, like they were plotted on a grid.

Icon style

Use consistent stroke weight, corner radius, and geometry so the set feels unified. Avoid fussy detail; think "panel label", not illustration.

Library preview

Preview the core set: navigation, actions, media controls, and status marks. If it belongs on a remote control or mixing board, it belongs here.

Sizing and usage

Snap icons to a pixel grid and use a small set of sizes (e.g., 16/20/24). Keep alignment consistent so icons don't look like they're drifting between frames.

Usage examples

Use icons with labels for clarity in primary actions, and icon-only buttons only when the meaning is universal. In dense UIs, treat icons like shorthand, not decoration.

Accessibility

Provide text alternatives (visible labels or aria-label), keep tap targets generous, and maintain contrast. If it's not readable on a dim CRT, it's not readable.
