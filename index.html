rationale.sens

This design system draws from early internet structures, brutalist design principles, and retro computing interfaces to create a system-oriented user experience rather than a conventional modern website.
The layout intentionally references Web 1.0-era design, using table-based structures, visible navigation, and minimal styling to emphasize clarity and hierarchy over decoration. Early websites often prioritized function and direct access to information, which is reflected in the raw, link-driven interface.
This approach overlaps with neo-brutalist web design, a style characterized by stark layouts, exposed structure, and a rejection of overly polished user interfaces. Brutalist websites often favor clarity and honesty in design, presenting content in a direct and unembellished way.
Additionally, the interface mimics a retro computing or terminal-like system, where content is organized as if it were part of a file directory or operating system. This creates a sense of interaction with a system rather than a traditional website, reinforcing the idea of navigation as exploration rather than guided browsing.
The design intentionally avoids modern UI conventions in favor of a more structured, mechanical, and system-driven aesthetic, blending nostalgia with contemporary brutalist principles.
